# Prompt

A prompt is the input given to an AI model. It tells the model what task to perform and what kind of response is expected.

The quality of the prompt often determines the quality of the output. Clear prompts help the model understand the task and reduce ambiguity.

A common misconception is that longer prompts are always better. In practice, prompts should be as simple as possible while still providing the necessary context.

For example:
- Less effective: `Please carefully explain all aspects of TCP networking in a way that is easy for beginners to understand.`
- Better: `Explain TCP for beginners.`

## Prompt Types

In a modern AI application, the model does not receive only the current user input. Instead, it processes a collection of prompts that together form the conversation context.

**System Prompt** defines the model's role, behavior, and constraints. It is typically provided by the application and has the highest priority.

Example: 
```shell
System: You are a Linux expert. Provide concise technical explanations.
```

**User Prompt** contains the task or question from the user.

Example:
```shell
User: How does a Linux namespace work?
```

**Assistant Prompt** consists of previous responses generated by the model. Together with earlier user messages, it forms the conversation history and helps maintain context across multiple turns.

Example:
```shell
User: Explain Docker.
Assistant: Docker is a container platform...
```

A simplified prompt structure looks like this:

```
System Prompt + Conversation History (User Prompt #1 + Assistant Prompt #1 + ... + User Prompt #N + Assistant Prompt #N) + Current User Prompt
```

When instructions conflict, the model follows higher-priority prompts first. In general, System Prompts override User Prompts, while previous conversation history provides additional context but has lower priority.

## Prompt Engineering

Prompt Engineering is the practice of designing prompts to obtain better results from AI models. It improves output quality without modifying the model itself.

Common techniques include:

- **Direct instructions**: clearly state the task.
- **Role prompting**: assign a role to the model. 
- **Output constraints**: specify the format, length, or structure.
- **Few-shot prompting**: provide examples of the desired output.
- **Step-by-step guidance**: break complex tasks into smaller stages.

Examples:
```
You are a senior Linux engineer.
Explain cgroups to a junior developer.
Summarize the article in three bullet points.
Return the result as a Markdown table.
```


## Test Prompt

```python
from transformers import AutoTokenizer, AutoModelForCausalLM
import torch

model_path = "./Qwen3.5-0.8B"

# 1. Load Tokenizer and Model
print("Loading model and tokenizer...")
tokenizer = AutoTokenizer.from_pretrained(model_path, local_files_only=True)
model = AutoModelForCausalLM.from_pretrained(
    model_path, local_files_only=True, device_map="cpu", torch_dtype="auto"
)

# 2. Define multiple prompt testing scenarios
test_cases = {
    "1. Basic Prompt (Simple Question)": [
        {"role": "user", "content": "How to learn Java?"}
    ],
    "2. System Prompt (Set Role)": [
        {"role": "system", "content": "You are a patient Java teacher for beginners."},
        {"role": "user", "content": "How to learn Java?"}
    ],
    "3. Constrained Prompt (Step-by-Step)": [
        {"role": "system", "content": "You are a clear programming tutor."},
        {"role": "user", "content": "Teach me Java basics. Use 3 simple steps, under 50 words."}
    ],
    "4. Few-Shot Prompt (Learn by Example)": [
        {"role": "system", "content": "You explain code with super simple analogies."},
        {"role": "user", "content": "What is a variable?"},
        {"role": "assistant", "content": "A box to store data."},
        {"role": "user", "content": "What is a loop?"}
    ],
    "5. Advanced Combined Prompt (Role + Rules + Task)": [
        {"role": "system", "content": "You are a fun Java coach. Keep it short, no jargon."},
        {"role": "user", "content": "Give a beginner tip to avoid Java bugs."}
    ]
}

# 3. Run streamlined testing loop
for title, messages in test_cases.items():
    print(f"\n=== Running: {title} ===")

    # Format the prompt into a standard PyTorch dictionary wrapper
    inputs = tokenizer.apply_chat_template(
        messages, return_tensors="pt", return_dict=True, add_generation_prompt=True
    ).to(model.device)

    # Generate content
    with torch.no_grad():
        outputs = model.generate(
            **inputs, # Unpacking dictionary handles inputs perfectly
            max_new_tokens=150,
            pad_token_id=tokenizer.eos_token_id,
            temperature=0.7,
            do_sample=True
        )

    # Slice outputs to decode ONLY the newly generated tokens
    new_tokens = outputs[0][inputs["input_ids"].shape[1]:]
    response = tokenizer.decode(new_tokens, skip_special_tokens=True)

    print(f"AI Response:\n{response.strip()}")
```
